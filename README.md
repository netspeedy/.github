Netspeedy is Simon Oakes' public infrastructure and platform engineering
umbrella. It brings together practical work across Linux systems, networking,
Ansible, Kubernetes, GitOps, CI/CD, DNS, certificates, self-hosted services and
operator-focused tooling.

The profile in this repository presents Netspeedy on GitHub: what the
organization represents, how the engineering approach works, and where public
examples of that work can be found.

## Netspeedy

Netspeedy represents production-minded infrastructure engineering rather than
isolated demo content. The focus is on systems that can be understood,
maintained, automated and recovered over time.

| Area | Focus |
| --- | --- |
| Infrastructure | Linux systems, networking, storage, DNS, certificates and service operations. |
| Automation | Ansible, GitHub Actions, scripts and repeatable workflows that reduce manual drift. |
| Platform engineering | Kubernetes, GitOps, CI/CD and platform services used where they improve operations. |
| Engineering practice | Source-controlled intent, reviewable change, clear ownership and recovery discipline. |
| Public work | Selected sites and tools that show practical operator-focused engineering. |

## KuberTech

KuberTech is the Kubernetes and GitOps platform slice of Netspeedy. It gives the
public profile a visible platform engineering thread while Netspeedy remains the
broader infrastructure and automation identity.
